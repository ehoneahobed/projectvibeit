# Day 2: Design and Wireframing

Today we'll transform your business concept into a visual design plan. We'll create wireframes, establish your design system, and plan the visual hierarchy of your website. This foundation will guide all your styling decisions in the coming days.

## Step 1: Create Wireframes

### What are Wireframes?

Wireframes are simple, black-and-white sketches that show the layout and structure of your website without colors, images, or detailed styling. They help you plan the user experience and content organization.

### Wireframing Tools

**Free Options:**
- **Figma** (recommended) - Professional design tool with free tier
- **Canva** - Easy-to-use design platform
- **Pen and Paper** - Quick sketching for initial ideas
- **Balsamiq** - Dedicated wireframing tool

## Step 2: Design System Planning

### Color Palette

Choose 3-5 colors for your brand:

#### Primary Colors (2-3 colors)
- **Primary**: Main brand color (used for buttons, links)
- **Secondary**: Supporting color (used for accents)
- **Accent**: Highlight color (used sparingly for emphasis)

#### Neutral Colors (2-3 colors)
- **Text**: Dark color for main text
- **Background**: Light color for backgrounds
- **Gray**: Medium gray for secondary text

#### Color Psychology by Business Type

**Coffee Shop**: Warm browns, creams, dark greens
**Photographer**: Blacks, whites, subtle grays
**Consultancy**: Blues, grays, professional tones
**Bakery**: Warm pinks, creams, chocolate browns

### Typography

Choose 2-3 fonts for your website:

#### Font Categories
- **Heading Font**: Display or serif font for titles
- **Body Font**: Sans-serif font for readable text
- **Accent Font**: Optional decorative font for special elements

#### Recommended Font Combinations

**Professional/Modern:**
- Headings: Inter, Roboto, or Open Sans
- Body: Inter, Roboto, or Open Sans

**Friendly/Approachable:**
- Headings: Poppins, Nunito, or Quicksand
- Body: Inter, Roboto, or Open Sans

**Luxury/Elegant:**
- Headings: Playfair Display, Merriweather, or Lora
- Body: Inter, Roboto, or Open Sans

## Step 3: Component Library

### Design Components

Plan these reusable components:

#### Buttons
- **Primary Button**: Main call-to-action

#### Cards
- **Service Card**: Display services/features
- **Testimonial Card**: Customer reviews

#### Forms
- **Input Fields**: Text, email, textarea
- **Form Layout**: Single column vs multi-column
- **Validation States**: Success, error, loading

## Step 4: Responsive Breakpoints

### Mobile-First Breakpoints

Plan your responsive design breakpoints:

```css
/* Mobile First */
/* Base styles for mobile (320px+) */

/* Tablet */
@media (min-width: 768px) {
  /* Tablet styles */
}

/* Desktop */
@media (min-width: 1024px) {
  /* Desktop styles */
}

/* Large Desktop */
@media (min-width: 1200px) {
  /* Large desktop styles */
}
```

### Layout Changes by Breakpoint

#### Mobile (320px+)
- Single column layout
- Stacked navigation
- Full-width sections
- Large touch targets

#### Tablet (768px+)
- Two-column layouts possible
- Horizontal navigation
- Side-by-side content
- Medium-sized elements

#### Desktop (1024px+)
- Multi-column layouts
- Complex navigation
- Sidebar options
- Hover effects

## Step 5: Visual Hierarchy

### Information Architecture

Plan how users will scan your content:

1. **Hero Section**: Most important message
2. **Services/Features**: What you offer
3. **About**: Why choose you
4. **Contact**: How to get in touch

### Visual Weight

Use size, color, and spacing to guide attention:

- **Large, bold headlines** for main messages
- **Medium-sized subheadings** for sections
- **Small, regular body text** for details
- **Color accents** for important elements

## Step 6: Accessibility Considerations

### Design for Everyone

Plan accessibility from the start:

#### Color Contrast
- Ensure sufficient contrast between text and background
- Don't rely solely on color to convey information
- Test with color blindness simulators

#### Typography
- Use readable font sizes (minimum 16px for body text)
- Maintain good line height (1.5-1.6 for body text)
- Choose fonts with good character distinction

#### Navigation
- Ensure keyboard navigation works
- Provide clear focus indicators
- Make touch targets large enough (minimum 44px)
